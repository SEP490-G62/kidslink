# UC-T-007: Xem và xử lý đơn xin phép của phụ huynh

## Tiếng Việt
- **Diễn giải:** Giáo viên có thể xem và xác nhận các đơn xin phép (nghỉ học, về sớm, ...) do phụ huynh gửi lên.
- **Diễn viên chính:** Giáo viên
- **Diễn viên phụ:** Hệ thống, Phụ huynh
- **Mô tả:**
  - Xem danh sách đơn xin phép của học sinh trong lớp.
  - Xem chi tiết từng đơn.
  - Xác nhận đã đọc hoặc chuyển tiếp cho nhà trường (nếu cần).
- **Điều kiện tiên quyết:**
  - Giáo viên đã đăng nhập hệ thống.
- **Kết quả sau cùng:**
  - Đơn xin phép được xác nhận đã đọc hoặc chuyển tiếp đúng quy trình.
- **Luồng chính:**
  1. Giáo viên truy cập mục "Đơn".
  2. Hệ thống hiển thị danh sách đơn xin phép của học sinh.
  3. Giáo viên xem chi tiết từng đơn.
  4. Giáo viên xác nhận đã đọc hoặc chuyển tiếp đơn cho nhà trường (nếu cần).
  5. Hệ thống cập nhật trạng thái đơn.
- **Luồng thay thế:**
  - Không có đơn nào: Hệ thống hiển thị thông báo "Không có đơn xin phép".
  - Lỗi xử lý đơn: Hệ thống báo lỗi và giữ nguyên trạng thái.

## English
- **Description:** Teacher can view and process leave requests (absences, early leave, etc.) submitted by parents.
- **Primary Actor:** Teacher
- **Secondary Actors:** System, Parent
- **Details:**
  - View list of leave requests for students in the class.
  - View details of each request.
  - Confirm as read or forward to the school (if needed).
- **Preconditions:**
  - Teacher is logged in.
- **Postconditions:**
  - Leave requests are confirmed as read or forwarded as required.
- **Normal Flow:**
  1. Teacher accesses the "Requests" section.
  2. System displays the list of leave requests for students.
  3. Teacher views details of each request.
  4. Teacher confirms as read or forwards the request to the school (if needed).
  5. System updates the request status.
- **Alternative Flows:**
  - No requests: System shows message "No leave requests".
  - Error processing request: System shows error and keeps the current state.
