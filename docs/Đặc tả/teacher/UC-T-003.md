# UC-T-003: Ghi nhận điểm danh học sinh

## Tiếng Việt
- **Diễn giải:** Giáo viên thực hiện điểm danh học sinh trong lớp vào đầu buổi học.
- **Diễn viên chính:** Giáo viên
- **Diễn viên phụ:** Hệ thống
- **Mô tả:**
  - Xem danh sách học sinh cần điểm danh.
  - Chọn trạng thái điểm danh cho từng học sinh (có mặt, vắng, có phép, không phép).
  - Lưu kết quả điểm danh.
- **Điều kiện tiên quyết:**
  - Giáo viên đã đăng nhập hệ thống.
- **Kết quả sau cùng:**
  - Kết quả điểm danh được lưu và hiển thị chính xác.
- **Luồng chính:**
  1. Giáo viên truy cập mục "Điểm danh".
  2. Hệ thống hiển thị danh sách học sinh cần điểm danh.
  3. Giáo viên chọn trạng thái điểm danh cho từng học sinh.
  4. Giáo viên lưu kết quả điểm danh.
  5. Hệ thống xác nhận lưu thành công.
- **Luồng thay thế:**
  - Lỗi lưu điểm danh: Hệ thống báo lỗi và giữ nguyên trạng thái.

## English
- **Description:** Teacher takes attendance for students at the beginning of the class.
- **Primary Actor:** Teacher
- **Secondary Actor:** System
- **Details:**
  - View list of students for attendance.
  - Select attendance status for each student (present, absent, excused, unexcused).
  - Save attendance results.
- **Preconditions:**
  - Teacher is logged in.
- **Postconditions:**
  - Attendance results are saved and displayed accurately.
- **Normal Flow:**
  1. Teacher accesses the "Attendance" section.
  2. System displays the list of students for attendance.
  3. Teacher selects attendance status for each student.
  4. Teacher saves the attendance results.
  5. System confirms successful save.
- **Alternative Flows:**
  - Error saving attendance: System shows error and keeps the current state.
